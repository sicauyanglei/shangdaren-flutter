## 1. 游戏概述

### 1.1 游戏类型
- 三人对战麻将类游戏
- 使用特殊的字牌进行

### 1.2 游戏目标
通过摸牌、出牌、吃碰招胡等操作，最先胡牌获胜

### 1.3 游戏特色
- 每人最多13张手牌
- 庄家14张
- 支持吃、碰、招（杠）、胡等操作
- 支持飘分系统

---

## 2. 牌具介绍

### 2.1 字牌组成

游戏使用24张字牌，共8个句子，每个句子3个字（每个字4张）：

| 句子序号 | 字牌 | 说明 |
|----------|------|------|
| 1句 | 上、大、人 | 首字"上"为特殊字 |
| 2句 | 丘、乙、己 | |
| 3句 | 化、三、千 | |
| 4句 | 七、十、土 | |
| 5句 | 尔、小、生 | |
| 6句 | 八、九、子 | |
| 7句 | 佳、作、亡 | |
| 8句 | 福、禄、寿 | 首字"福"为特殊字 |

### 2.2 牌型结构

每张字牌包含以下属性：

| 属性 | 说明 | 取值范围 |
|------|------|----------|
| character | 字本身 | 上、大、人... |
| sentence | 句子编号 | 1-8 |
| position | 位置 | 0（首字）、1（中字）、2（尾字） |

### 2.3 特殊牌

- **上**（句子1的首字）：具有特殊地位
- **福**（句子8的首字）：具有特殊地位

### 2.4 牌堆数量
- 总共 24 × 4 = 96 张牌

## 3. 游戏流程

### 3.1 大厅阶段

1. 玩家登录后进入大厅
2. 默认显示8张牌桌（可扩展至100张）
3. 点击房间可查看详情或加入

### 3.2 房间阶段

1. 玩家进入房间后成为旁观者
2. 点击"加入房间"正式入座
3. 点击"准备"表示就绪
4. 3人全部准备后游戏自动开始

### 3.3 飘分阶段

游戏开始前，每位玩家需要选择飘分：

| 飘分选项 | 说明 |
|----------|------|
| 0分 | 不飘 |
| 1分 | 飘1分 |
| 2分 | 飘2分 |
| 3分 | 飘3分 |

- 飘分选择完成后，进入发牌阶段
- 如果玩家超时未选择飘分，自动飘0分

### 3.4 发牌阶段

- **庄家**：获得14张牌
- **闲家**：各获得13张牌
- 剩余牌堆放在桌面中央

### 3.5 出牌阶段

游戏进行多轮，每轮流程：

```
┌─────────────────────────────────────┐
│  1. 摸牌：玩家从牌堆摸取1张牌        │
│  2. 行动：可选择吃/碰/招/胡（如有）  │
│  3. 出牌：将不需要的1张牌打出         │
│  4. 轮换：下一位玩家继续             │
└─────────────────────────────────────┘
```

### 3.6 回合超时

- 每个回合限时 **30秒**
- 超时自动出牌（随机选择一张）

## 4. 操作规则

### 4.1 吃牌（Chi）

**触发条件**：
- 必须是**上家**打出的牌
- 手牌中已有同句的**2张牌**（除首字外）
- 手牌中不能已有吃成的句子

**吃牌逻辑**（代码实现）：

```javascript
function canPlayerChi(player, card) {
  // 已吃过则不能再吃
  if (player.melds.some(m => m.type === 'sequence')) return false;

  // 手牌中必须有同句的牌
  const sentenceCards = player.hand.filter(c => c.sentence === card.sentence);
  if (sentenceCards.length < 2) return false;

  // 检查是否能组成句子
  const positions = new Set(sentenceCards.map(c => c.position));

  // 首字只能与中尾字配合
  if (card.position === 0) return positions.has(1) && positions.has(2);
  // 中字只能与首尾字配合
  if (card.position === 1) return positions.has(0) && positions.has(2);
  // 尾字只能与首中字配合
  if (card.position === 2) return positions.has(0) && positions.has(1);

  return false;
}
```

**吃牌示例**：

| 场景 | 手牌 | 上家出牌 | 可吃成 |
|------|------|----------|--------|
| 例子1 | 化、三 | 三 | 化、三、千（3句） |
| 例子2 | 七、十 | 七 | 七、十、土（4句） |

**规则**：
- 吃牌后，该玩家继续出牌（不影响出牌顺序）
- 吃成的句子显示为"倒牌"

### 4.2 碰牌（Peng）

**触发条件**：
- 必须是**其他玩家**打出的牌
- 手牌中已有该字的**2张**

**碰牌逻辑**（代码实现）：

```javascript
function canPlayerPeng(player, card) {
  // 已碰过（三张）则不能再碰
  if (player.melds.some(m => m.type === 'triplet')) return false;

  // 手牌中必须有2张相同的字
  const count = player.hand.filter(c => c.character === card.character).length;
  return count >= 2;
}
```

**规则**：
- 碰牌后，该玩家继续出牌
- 碰牌优先级**高于**吃牌
- 碰成的刻子显示为"倒牌"

### 4.3 招牌/杠（Zhao）

**触发条件**：
- 手牌中有**3张**相同的字
- 未招过该字的杠

**招牌逻辑**（代码实现）：

```javascript
function canPlayerZhao(player, card) {
  // 已招过（四张）则不能再招
  if (player.melds.some(m => m.type === 'quartet')) return false;

  // 手牌中必须有3张相同的字
  const count = player.hand.filter(c => c.character === card.character).length;
  return count >= 3;
}
```

**规则**：
- 招牌后，从牌堆补1张牌
- 招牌后，该玩家继续出牌
- 招牌优先级**高于**碰牌和吃牌
- 招成的杠显示为"倒牌"

### 4.4 操作优先级

当多位玩家可响应时，优先顺序为：

```
胡 > 招 > 碰 > 吃
```

### 4.5 操作按钮

游戏界面中的操作按钮：

| 按钮 | 颜色 | 说明 |
|------|------|------|
| 吃 | 绿色 | 可吃牌时显示 |
| 碰 | 蓝色 | 可碰牌时显示 |
| 招 | 紫色 | 可招牌时显示 |
| 胡 | 红色 | 可胡牌时显示 |

## 5. 胡牌规则

### 5.1 胡牌条件

**当前实现**（简化版）：

```javascript
function canPlayerHu(player, card) {
  // 将card加入手牌
  const hand = card ? [...player.hand, card] : [...player.hand];

  // 手牌数达到14（或者13+1）即可能胡
  if (hand.length < 14) return false;

  return true;
}
```

### 5.2 胡牌方式

| 类型 | 说明 | 代码标识 |
|------|------|----------|
| 自摸 | 摸牌时直接胡牌 | `isZimo: true` |
| 胡牌 | 其他玩家打出可胡的牌 | `isZimo: false` |

### 5.3 胡牌流程

```javascript
function executeHu(room, playerIndex, isZimo, card) {
  room.game.status = 'roundEnd';
  clearTurnTimer(room.id);

  // 计算分数
  // ...

  // 广播胡牌消息
  broadcastToRoom(room, {
    type: 'game:hu',
    playerIndex: playerIndex,
    isZimo: isZimo,
    card: card,
    settlements: settlements
  });

  // 3秒后进入下一局
  setTimeout(() => { endRound(room); }, 3000);
}
```

### 5.4 听牌

**触发条件**：
- 只剩1张牌可胡时自动进入听牌状态
- 系统显示可以胡的牌

## 6. 特殊规则

### 6.1 句满

当某一句的所有字牌（12张）全部被碰、招、吃或胡走后，该句称为"句满"。

### 6.2 倒牌

玩家吃、碰、招后亮出的牌称为"倒牌"：

| 类型 | 英文 | 说明 |
|------|------|------|
| 句子 | sequence | 吃牌后亮出 |
| 刻子 | triplet | 碰牌后亮出 |
| 杠子 | quartet | 招牌后亮出 |

### 6.3 流局

当牌堆摸完仍未有人胡牌时，称为流局：

```javascript
function handleLiuJu(room) {
  room.game.status = 'roundEnd';
  clearTurnTimer(room.id);

  broadcastToRoom(room, {
    type: 'game:liuju',
    deckCount: 0
  });

  setTimeout(() => { endRound(room); }, 3000);
}
```

### 6.4 游戏结束

- 游戏共 **8局**
- 8局结束后显示最终排名

```javascript
if (room.game.roundNumber > 8) {
  // 8局结束，游戏结束
  room.status = 'waiting';
  room.game = null;

  // 计算最终得分并广播
}
```

## 7. 计分规则

### 7.1 飘分影响

每位玩家的飘分将计入最终得分。

### 7.2 结算流程

1. 胡牌后计算各玩家得分
2. 每局结束后显示各玩家得分
3. 8局结束后显示最终排名

### 7.3 得分计算

当前实现中，胡牌者获得胜利，其他玩家得0分。

## 9. 登录系统

### 9.1 手机号登录

1. 输入手机号（需符合1开头11位格式）
2. 点击"发送验证码"
3. 输入验证码（开发环境验证码显示在服务器日志 `/api/logs`）
4. 登录成功跳转大厅

### 9.2 验证码规则

- 6位数字
- 有效期60秒
- 同一手机号60秒内不能重复发送

### 9.3 游客登录

点击"游客登录(不保存进度)"可快速进入游戏：
- 自动生成游客名称
- 不保存游戏记录
- token为`guest_token`开头

### 9.4 登录状态

登录成功后存储：
- `auth_token`：认证令牌
- `player_id`：玩家ID
- `player_name`：玩家名称
- `player_phone`：手机号（如通过手机登录）

## 10. 网络配置

### 10.1 服务端口

| 服务 | 端口 | 说明 |
|------|------|------|
| HTTP | 8081 | Web服务器和API |
| WebSocket | 8080 | 实时游戏通信 |

### 10.2 API 端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/player/send-sms` | POST | 发送验证码 |
| `/api/player/login` | POST | 手机号登录 |
| `/api/player/logout` | POST | 退出登录 |
| `/api/player/online` | GET | 获取在线玩家列表 |
| `/api/rooms` | GET | 获取所有房间信息 |
| `/api/logs` | GET | 获取服务器日志（开发用） |

### 10.3 WebSocket 消息

| 消息类型 | 方向 | 说明 |
|----------|------|------|
| `auth` | 客户端→服务器 | 认证玩家 |
| `room:list` | 服务器→客户端 | 房间列表 |
| `room:join` | 客户端→服务器 | 加入房间 |
| `room:joined` | 服务器→客户端 | 加入成功 |
| `room:ready` | 客户端→服务器 | 玩家准备 |
| `room:update` | 服务器→客户端 | 房间状态更新 |
| `room:game:start` | 服务器→客户端 | 游戏开始 |
| `room:leave` | 客户端→服务器 | 离开房间 |
| `room:left` | 服务器→客户端 | 离开成功 |
| `room:heartbeat` | 双向 | 心跳保活 |

### 10.4 游戏消息

| 消息类型 | 方向 | 说明 |
|----------|------|------|
| `game:action` | 客户端→服务器 | 玩家操作（出牌/吃/碰/招/胡） |
| `game:action:result` | 服务器→客户端 | 操作结果 |
| `game:action:request` | 服务器→客户端 | 请求玩家响应 |
| `game:hu` | 服务器→客户端 | 胡牌广播 |
| `game:liuju` | 服务器→客户端 | 流局广播 |
| `game:player:disconnect` | 服务器→客户端 | 玩家断线 |
